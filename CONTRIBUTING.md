# 🤝 Contributing

Спасибо за желание обновить доку! Этот репозиторий — неофициальная рабочая
документация по разработке модулей для [Heroku Userbot](https://github.com/coddrago/Heroku).

## 📌 Главное правило

> Вся информация в `docs/` должна быть **сверена с исходным кодом ядра**
> (`source/` или свежий клон `coddrago/Heroku`). Если в дока-сайте и в коде
> расхождения — код приоритетнее (сайт может отставать от релизов).

## 🛠 Как обновить доку

1. **Форкни → ветка.** Создай ветку `docs/update-<topic>`, например
   `docs/update-watchers`.
2. **Сверься с исходниками.** Открой соответствующий файл в `source/`
   (`loader.py`, `types.py`, `modules_loader.py` …) и сверь сигнатуры/поведение.
3. **Редактируй только `docs/` и `examples/`.** Папка `source/` — первоисточник,
   её не правим (она чит-онли снапшот ядра).
4. **Один PR — одна тема.** Меняй декораторы в одном PR, валидаторы — в другом.
5. **Примеры должны работать.** Если добавляешь модуль в `examples/`, он должен
   загружаться через `.dlmod` без правок.
6. **Стиль:** русский язык, заголовки через `#`/`##`, таблицы для референсов,
   блоки кода с подсветкой `python`.

## ✅ Чеклист перед PR

- [ ] Информация сверена с `source/` (или свежим клоном ядра)
- [ ] Нет выдуманных команд / декораторов / хуков
- [ ] Примеры (если правишь `examples/`) синтаксически валидны
- [ ] Ссылки в README живые
- [ ] Язык/орфография вычитаны

## 🧪 Как проверить модуль локально

```bash
git clone https://github.com/coddrago/Heroku
cd Heroku && python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
python3 -m heroku
```

Затем в юзерботе:
```
.dlmod https://github.com/krutyshkin/heroku-modbook/blob/main/examples/hello.py
```
