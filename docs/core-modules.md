## APILimiter — защита от спама вызовов к API

**Команд: 2**

| Команда | Описание |
|---|---|
| `.api_fw_protection` | Включает/отключает защиту API от флуда. Открывает инлайн с подтверждением. Когда активна — перехватывает все исходящие запросы и замедляет при превышении порога. |
| `.suspend_api_protect <секунды>` | Временно приостанавливает защиту (api_fw) на N секунд. |

**Конфиг (`.cfg APILimiter`)**

| Ключ | Описание |
|---|---|
| `time_sample` | Время в секундах для подсчёта запросов (по умолч. 15) |
| `threshold` | Макс. количество запросов до срабатывания (по умолч. 100) |
| `local_floodwait` | Пауза в секундах при срабатывании (10–3600, по умолч. 30) |
| `forbidden_methods` | Список запрещённых методов API |

## Evaluator — выполнение кода (Python, C, C++, Node.js, Go, Rust)

**Команд: 6**

| Команда | Описание |
|---|---|
| `.e <код>` (алиас `.eval`) | Выполняет Python-код в контексте юзербота. Можно ответить на сообщение — его текст станет кодом. Автоматически цензурирует чувствительные данные (токены, session, номер телефона). |
| `.ecpp <код>` | Компилирует и выполняет C++ (требует `g++`). |
| `.ec <код>` | Компилирует и выполняет C (требует `gcc`). |
| `.enode <код>` | Выполняет JS через Node.js (требует `node`). |
| `.eg <код>` | Выполняет Go-код. |
| `.ers <код>` | Выполняет Rust-код. |

## Help — справка по модулям и командам

**Команд: 3**

| Команда | Описание |
|---|---|
| `.help [название]` | Список всех модулей и команд. Без аргументов — общий список; с именем — подробная справка. Флаги: `-f` (все команды, включая недоступные по правам), `-c` (только core-модули), `-l` (только загруженные внешние). |
| `.helphide <модуль ...>` | Скрывает/показывает модули в `.help` (тогл). |
| `.support` | Ссылка на официальный чат поддержки Heroku. |

**Конфиг (`.cfg Help`)**

| Ключ | Описание |
|---|---|
| `core_emoji` | Эмодзи для core-модулей в списке |
| `plain_emoji` | Эмодзи для внешних модулей в списке |
| `banner_url` | Ссылка на баннер в `.help` |
| `show_preview_in_help` | Показывать превью при `.help [модуль]` |

## HerokuBackup — резервное копирование и восстановление

**Команд: 7**

| Команда | Описание |
|---|---|
| `.backupall` | Бэкап БД + всех модулей + pip-зависимостей. Сохраняется в топик Backups в формате `.backup`. |
| `.backupdb` | Бэкап базы данных (файл в топик Backups). |
| `.backupmods` | Бэкап всех загруженных модулей (zip). |
| `.restoreall` *(ответ на .backup)* | Восстановление из `.backup`. После — авто-перезапуск. |
| `.restoredb` *(ответ на json)* | Восстановление БД. |
| `.restoremods` *(ответ на zip)* | Восстановление модулей. |
| `.set_backup_period <часы>` | Период авт. резервного копирования (0–199). |

**Команд: 2**

|---|---|
| `.config [модуль] [флаг]` (алиас `.cfg`) | Открывает конфигуратор модулей через инлайн. |

| `quote_media` | Сделать баннер в виде цитаты |
| `invert_media` | Инвертировать отображение медиа |

## HerokuSecurity — права доступа к модулям/командам

**Команд: 14**

| Команда | Описание |
|---|---|
| `.security [команда]` | Меню управления правами. Без аргументов — инлайн-меню; с именем команды — права для конкретной. |
| `.inlinesec [инлайн-команда]` | Аналог `.security`, но для инлайн-команд бота. |
| `.querysec` | Управление разрешением использовать инлайн-запросы (@бот в любом чате). |
| `.owneradd <@user> / <id>` | Даёт полные права на управление юзерботом. Флаги: `-f` (без подтверждения), `-n` (добавить в nonick). |
| `.ownerrm <@user> / <id>` | Удаляет пользователя из списка owner. |
| `.ownerlist` | Список всех пользователей с доступом. |
| `.newsgroup <имя>` | Создаёт новую security group. |
| `.sgroups` | Список всех security groups с числом участников и правил. |
| `.sgroup <имя>` | Подробности группы: участники и их права. |
| `.delsgroup <имя>` | Удаляет security group. |
| `.sgroupadd <группа> <@user> / <id>` | Добавляет пользователя в группу. |
| `.sgroupdel <группа> <@user> / <id>` | Удаляет пользователя из группы. |
| `.tsec <команда/модуль> [@user/chat]` | Временное право на команду/модуль. Длительность: `1d`, `2h`, `30m`, `60s`. Без — навсегда. |
| `.tsecrm <sec-правило>` | Удаляет временное правило. |
| `.tsecclr` | Очищает все временные правила сразу. |

## HerokuSettings — настройки юзербота

**Команд: 10**

| `.nonickusers` | Список nonick-пользователей. |
| `.nonickchat [chat_id]` | Добавляет/убирает текущий чат из nonick. |
| `.nonickchats` | Список nonick-чатов. |
| `.nonickcmd <команда>` | Добавляет/убирает команду из nonick. |
| `.nonickcmds` | Список nonick-команд. |

## Settings — системные настройки

**Команд: 14**

| Команда | Описание |
|---|---|
| `.heroku` | Версия, последний коммит, версия библиотеки, разработчики, ветка. |
| `.setprefix <префикс>` | Новый префикс команд (по умолч. `.`). |
| `.aliases` | Список всех алиасов. |
| `.addalias <алиас> <команда>` | Создаёт алиас. Несколько через `&&`. Пример: `.addalias ub ubinfo`. |
| `.delalias <алиас>` | Удаляет алиас. |
| `.blacklist` | Список чатов в чёрном списке (бот игнорирует команды оттуда). |
| `.unblacklist <chat_id>` | Убирает чат из чёрного списка (ID или ответ на сообщение). |
| `.blacklistuser <@user> / <id>` | Добавляет пользователя в чёрный список. |
| `.unblacklistuser <@user> / <id>` | Убирает пользователя из чёрного списка. |
| `.cleardb` | Полностью очищает базу данных. |
| `.clearmodule <модуль>` | Удаляет все записи БД конкретного модуля (сброс настроек и данных). |
| `.togglecmd <модуль> <команда>` | Включает/отключает конкретную команду модуля. |
| `.togglemod <модуль>` | Включает/отключает модуль целиком. |
| `.installation` | Гайд по установке Heroku на разные платформы. |
