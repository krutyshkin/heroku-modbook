# 🌿 Модуль DigitalGarden — публикация заметок в веб

> 📍 [README](../README.md) → **Документация** → DigitalGarden
> · соседние: [Core Modules](core-modules.md) · [Кастомизация](customization.md)

> По официальной dev-доке `sunnexgb.github.io/Heroku-documentations-md`
> (Guide for DigitalGarden module).

DigitalGarden — модуль для публикации ваших заметок в веб (GitHub Pages).
Заметки деплоятся в ваш репозиторий с помощью GitHub Actions.

## Подготовка репозитория

Нужно создать свой репозиторий на GitHub, куда будут деплоиться все заметки.
Переходим на [SunnexGB/DigitalGarden](https://github.com/SunnexGB/DigitalGarden)
и следуем шагам:

1. Генерируем из шаблона репозиторий.
2. Пишем любое название (которое пропустит GitHub) и описание по желанию.
   Репозиторий должен быть **публичным**.
3. Нажимаем на **Settings** → **Pages**.
4. Выбираем способ билдинга и деплоя — **GitHub Actions**.

**Итог:** появится страница, но на ней ничего не будет (пока что).

## Настройка модуля

Чтобы публиковать заметки, нужно настроить модуль командой `.cfg digitalgarden`.
Поля, которые нужно настроить:

- `repo_name` — имя репозитория
- `username` — ваш GitHub-username
- `github_token` — токен доступа

### Получение `github_token`

1. Переходим на страницу создания токена (GitHub → Settings → Developer settings →
   Personal access tokens).
2. Указываем имя токена (ключа).
3. Выбираем `Only select repositories` и выбираем репозиторий, сгенерированный ранее.
4. Нажимаем `Add permissions` и выбираем 2 вещи:
   - **Contents** — обязательно выбрать `Read and write`
   - **Pull requests** — то же самое (`Read and write`)
5. Создаём токен и копируем его (скопировать можно только 1 раз).

Токен устанавливается командой:
```
.fcfg digitalgarden github_token ваш_токен
```

## Команды

| Команда | Описание |
|---|---|
| `.dg` | Опубликовать вашу заметку в веб. Поддерживает dg-теги как аргументы. Например, для главной страницы (куда кидает всегда) — тег `dg-home`. |
| `.dgc` | Управление статусом dg-тегов. Тогл-команда: отключает/включает указанные теги. |

## Список dg-тегов

> Какие-то из них модуль не проставляет сам — придётся указывать вручную.

| Свойство | Что делает |
|---|---|
| `dg-home` | Делает заметку главной страницей. Должно быть только у одной заметки. |
| `dg-show-local-graph` | Показывает локальный граф для заметки (даже если глобально выключен). |
| `dg-show-backlinks` | Показывает список обратных ссылок. |
| `dg-show-toc` | Показывает оглавление. |
| `dg-show-inline-title` | Показывает заголовок заметки сверху страницы. |
| `dg-show-filetree` | Показывает файловое дерево справа. |
| `dg-enable-link-preview` | Включает превью заметок при наведении на ссылки. |
| `dg-enable-search` | Включает поиск. |
| `dg-show-tags` | Показывает теги из frontmatter на странице. |
| `dg-pass-frontmatter` | Передаёт весь frontmatter в шаблон без фильтрации (для кастомных шаблонов). |
| `dg-permalink` | Задаёт собственный URL страницы. Например `articles/my-post`. |
| `dg-pinned` | Закрепляет заметку сверху. |
| `dg-hide` | Полностью скрывает заметку: из дерева, поиска, графа, sitemap, RSS и т.д., хотя файл остаётся опубликованным. |
| `dg-hide-in-graph` | Скрывает только из графа. |
| `dg-hide-in-filetree` | Скрывает только из файлового дерева, но оставляет доступной через поиск и ссылки. |
| `dg-metatags` | Задаёт SEO/OpenGraph-метатеги (`description`, `og:image`, `og:title` и др.). |
| `dg-content-classes` | Добавляет CSS-классы к странице для индивидуального оформления. |
| `dg-note-icon` | Назначает иконку заметке. |

> Больше тегов — в [официальной документации DigitalGarden](https://dg-docs.ole.dev/).
